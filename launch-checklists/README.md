# 📋 Open-Source Project Launch Checklists

This directory contains structured, actionable checklists to guide your core maintainers and contributors through a successful public project launch. 

---

## ⏳ Phase 1: Pre-Launch Checklist (T-Minus 7 Days)
Before telling the public or the media about your codebase, ensure your repository foundation is completely solid:

*   [ ] **Code Freeze:** Ensure the `main` branch is stable, passing all CI/CD test builds, and free of glaring bugs.
*   [ ] **Documentation Review:** Verify that installation, configuration, and getting-started guides are completely clear.
*   [ ] **License Verification:** Confirm that an open-source license file (like `LICENSE`) is active in the repository root.
*   [ ] **Media Training:** Ensure core maintainers have reviewed the resources in the `./media-training` directory.
*   [ ] **Target List Ready:** Finalize your list of tech journalists, bloggers, and newsletter creators to pitch.

---

## 🚀 Phase 2: Launch Day Checklist (The Big Day)
Execute these items in sequential order on the day you go public:

*   [ ] **Flip Repo to Public:** If your repository was private during development, change visibility settings to **Public**.
*   [ ] **Send Press Pitches:** Use the template in `./templates/media-pitch.md` to send individualized pitches to tech journalists.
*   [ ] **Post on Hacker News:** Submit your entry using the framework in `./templates/hacker-news-post.md` following "Show HN" rules.
*   [ ] **Social Media Blitz:** Coordinate core team members to post simultaneously across X/Twitter, LinkedIn, and Mastodon.
*   [ ] **Monitor Influx:** Keep an eye on GitHub Issues, pull requests, and community chat channels for traffic spikes.

---

## 📈 Phase 3: Post-Launch Checklist (The Next 48 Hours)
Capitalize on the launch momentum and keep your early community engaged:

*   [ ] **Triage Incoming Feedback:** Respond to every single GitHub Issue or discussion thread within a 2-hour window.
*   [ ] **Welcome Contributors:** Be welcoming to early Pull Requests, providing clear review feedback to encourage open collaboration.
*   [ ] **Track Metrics:** Monitor repository star counts, forks, clones, and website analytics to evaluate your pitch messaging.
*   [ ] **Follow-Up Pitches:** Send polite follow-up messages to journalists who opened your initial email but have not yet written a piece.
