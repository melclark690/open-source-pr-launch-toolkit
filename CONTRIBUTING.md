# Contributing to the Open-Source Tech PR Toolkit

Thank you for your interest in contributing to this project! We welcome contributions from public relations professionals, open-source maintainers, technical writers, and developers. 

By contributing, you help make the open-source ecosystem more accessible to media and communities globally.

## 🚀 How to Contribute

### 1. Suggesting Improvements or Reporting Bugs
If you find a typo, broken link, or think a template could be improved, please open an **Issue** on GitHub:
* Navigate to the **Issues** tab.
* Click **New Issue**.
* Describe the optimization or framework variation you would like to see.

### 2. Submitting Template Changes (Pull Requests)
If you want to directly add a new PR template, checklist, or communication framework:
1. **Fork** this repository to your own GitHub account.
2. Create a new branch for your feature (`git checkout -b feature/NewPRTemplate`).
3. Add your framework or copy edits to the relevant files.
4. **Commit** your changes with clear, descriptive commit messages.
5. **Push** the branch to your fork and submit a **Pull Request (PR)** to our `main` branch.

## 📋 Contribution Guidelines

* **Keep It Generic:** Ensure all templates, pitch notes, and framework materials are generalized. Do not include proprietary data, company names, or sensitive communication strategy metrics.
* **Developer-Centric:** Focus on copy and strategies that prioritize developer awareness, open-source licensing compliance, and technical transparency.
* **Accessible Language:** Use clear, professional, and accessible language to support global contributors and cross-functional teams.

## ⚖️ Code of Conduct
We expect all community contributors to maintain professional, respectful, and collaborative communication across all issues, pull requests, and discussions.
